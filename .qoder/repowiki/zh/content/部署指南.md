## 简介

本指南为基金管理系统提供全面的部署方案，涵盖从本地开发到生产环境的各种部署场景。该系统基于Spring Boot框架构建，采用Maven作为项目管理工具，前端使用React + Vite技术栈，具有轻量级特性和高度可扩展性。

**更新** 系统现已支持CLI和Web双模式部署，通过双JAR打包配置实现单一代码库的多模式运行能力。新增的cron作业安装脚本（scripts/install-cron.sh）提供了系统级的定时任务自动化部署能力，替代了原有的Spring @Scheduled注解方式，增强了任务调度的可靠性和自动化程度。

## 部署场景

### 本地开发环境部署

#### 开发环境准备

1. **环境要求**
   - Java 17或更高版本
   - Node.js 16+
   - Maven 3.6+ 
   - MySQL 8.0+

2. **后端项目克隆和构建**
   ```bash
   # 克隆项目
   git clone <repository-url>
   
   # 进入项目目录
   cd fund
   
   # 清理并构建后端项目（生成双JAR）
   ./mvnw clean package
   ```

3. **前端项目安装和构建**
   ```bash
   # 进入前端目录
   cd fund-web
   
   # 安装依赖
   npm install
   
   # 启动开发服务器
   npm run dev
   ```

4. **启动应用程序**
   ```bash
   # Web模式启动
   ./mvnw spring-boot:run
   
   # 或者使用Java直接运行Web模式
   java -jar target/fund-*-web.jar
   
   # CLI模式启动
   java -jar target/fund-*-cli.jar
   
   # Web模式启动（显式指定）
   java -jar target/fund-*-cli.jar --web
   ```

## 容器化部署

### Docker镜像构建

#### Dockerfile配置

```dockerfile

WORKDIR /app

# 复制JAR文件
COPY target/*.jar app.jar

# 暴露端口

# 启动命令
ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### 多阶段构建优化

```dockerfile
# 构建阶段
FROM maven:3.9.1-jdk-17 AS builder
WORKDIR /app
COPY pom.xml .
COPY src src
RUN mvn package -DskipTests

# 运行阶段
FROM openjdk:17-jre-slim
WORKDIR /app
COPY --from=builder target/*.jar app.jar

# 安全配置
RUN addgroup --system spring && \
    adduser --system spring && \
    chown -R spring:spring /app
USER spring:spring

EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```
