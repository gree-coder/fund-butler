## 一、产品概述

### 1.1 产品定位

「基金管家」是一款面向个人投资者的**基金管理与查询 Web 应用**，定位为"一站式基金数据聚合管理工具"。产品不涉及真实交易，专注于**基金数据展示、持仓管理、收益分析和投资决策辅助**，帮助用户高效管理分散在多个平台的基金投资。

### 1.2 产品愿景

让每一位基金投资者都能清晰掌握自己的投资全貌，做出更理性的投资决策。

### 1.3 竞品参考

| 竞品 | 核心特点 | 可借鉴之处 |
|------|---------|-----------|
| 养基宝 | 简洁高效、OCR识图录入、多平台持仓汇总、盈亏分析 | 极简交互、图片识别同步持仓 |
| 天天基金 | 数据全面、基金交易、基金经理数据 | 基金数据库完整度、筛选维度 |
| 蛋卷基金 | 指数估值、组合投资、智能定投 | 指数估值体系、资产配置展示 |
| 且慢 | 投资方案、长期投资理念 | 投资组合管理、策略展示 |

### 1.4 差异化价值

- **纯工具属性**：不做交易，不接触用户资金，零风险使用
- **Web 优先**：无需下载App，浏览器直接使用，跨设备同步
- **数据聚合**：汇总多平台持仓，一屏掌握投资全貌
- **智能分析**：提供专业级收益归因、风险分析和资产配置建议

## 二、目标用户

### 2.1 用户画像

| 维度 | 描述 |
|------|------|
| **年龄** | 25-45岁 |
| **职业** | 上班族、自由职业者、有一定理财意识的年轻人 |
| **投资经验** | 初级到中级基金投资者 |
| **痛点** | 基金分散在多个平台、难以汇总查看整体收益、缺乏专业分析工具 |
| **使用场景** | 工作间隙查看基金估值、周末复盘投资收益、调仓决策参考 |

### 2.2 典型用户故事

1. **小王（28岁，程序员）**：在支付宝、微信理财通、天天基金三个平台买了基金，想在一个地方看到所有持仓的总收益和每日盈亏。
2. **小李（35岁，产品经理）**：想对比不同基金的历史业绩、回撤、基金经理能力，辅助自己做调仓决策。
3. **小张（30岁，教师）**：定投了多只基金，希望看到定投收益曲线和年化收益率，评估自己的定投策略是否合理。

## 四、核心功能需求（按优先级排列）

### P0 - 必须实现（MVP）

#### 4.1 首页 Dashboard

**功能描述**：用户登录后的主页面，一屏展示投资全貌。

| 子功能 | 说明 |
|--------|------|
| 资产总览 | 展示总资产、总收益（金额+百分比）、今日收益 |
| 持仓列表 | 展示所有持仓基金，包含名称、代码、持有份额、最新净值、估算值、当日涨跌幅、持有收益 |
| 收益趋势 | 简易折线图展示近7天/30天收益走势 |
| 快捷操作 | 添加持仓、搜索基金的快捷入口 |

**交互要点**：
- 资产数据支持"隐藏金额"功能（隐私保护）
- 持仓列表按持有金额默认降序排列，支持按涨跌幅、收益率排序
- 下拉刷新更新估值数据

---

#### 4.2 基金查询与详情

**功能描述**：提供基金搜索和详细信息查看能力。

| 子功能 | 说明 |
|--------|------|
| 基金搜索 | 支持按基金名称、代码模糊搜索，实时联想 |
| 基金详情 | 展示基金基本信息、净值走势、历史业绩、风险指标 |
| 净值走势 | 可切换时间范围（近1月/3月/6月/1年/3年/全部），支持对比沪深300等基准 |
| 历史业绩 | 展示近1周/1月/3月/6月/1年/2年/3年/成立以来收益率 |
| 基金经理 | 展示基金经理姓名、从业年限、管理规模、历史业绩 |
| 持仓分析 | 展示十大重仓股、行业分布、资产配置比例 |
| 费率信息 | 展示申购费、赎回费、管理费、托管费 |

**交互要点**：
- 搜索框常驻顶部导航栏，全局可用
- 净值走势图支持手势拖动查看具体日期净值
- 可一键添加到自选/添加到持仓

---

#### 4.3 持仓管理

**功能描述**：管理用户的基金持仓数据。

| 子功能 | 说明 |
|--------|------|
| 手动添加 | 输入基金代码、买入金额/份额、买入日期、所属账户 |
| 持仓编辑 | 修改持仓信息、记录加仓/减仓/清仓操作 |
| 交易记录 | 按时间线展示每只基金的买卖记录 |
| 多账户 | 支持创建多个账户（如"支付宝"、"天天基金"等），按账户筛选持仓 |
| 资产分布 | 饼图展示持仓的基金类型分布（股票型/混合型/债券型/货币型/QDII等） |

**交互要点**：
- 添加持仓时，输入基金代码自动联想基金名称
- 支持批量操作（批量删除、批量移动账户）
- 清仓操作需二次确认

---

#### 4.4 自选基金

**功能描述**：关注但未购买的基金管理。

| 子功能 | 说明 |
|--------|------|
| 添加自选 | 通过搜索或基金详情页添加 |
| 自选列表 | 展示基金名称、代码、最新净值、日涨跌幅、近1月/3月/1年收益率 |
| 分组管理 | 支持自定义分组（如"待买入"、"观察中"） |
| 排序筛选 | 支持按涨跌幅、收益率等排序 |

---

### P1 - 重要功能

#### 4.5 收益分析

**功能描述**：提供多维度的投资收益分析。

| 子功能 | 说明 |
|--------|------|
| 收益曲线 | 可视化展示累计收益、日收益曲线，支持对比沪深300等基准 |
| 时段统计 | 按日/周/月/年统计收益，展示收益日历（类似GitHub热力图） |
| 年化收益 | 计算持有期年化收益率、时间加权收益率 |
| 回撤分析 | 展示最大回撤、当前回撤、回撤恢复时间 |
| 收益归因 | 按基金类型、行业、单只基金分析收益贡献 |
| 定期报告 | 自动生成周报/月报，总结本期收益、持仓变动、市场表现 |

**交互要点**：
- 收益曲线支持拖动查看具体日期数据
- 收益日历用颜色深浅表示盈亏幅度（绿色=盈利，红色=亏损）
- 报告支持导出为图片/PDF

| 子功能 | 说明 |

## 五、页面设计规范

### 5.1 设计原则

1. **简洁高效**：参考养基宝的极简设计理念，避免信息过载
2. **数据可视化优先**：用图表替代大段文字，让数据一目了然
3. **操作直觉化**：核心操作不超过3步完成
4. **响应式设计**：适配桌面端和移动端浏览器

### 5.2 页面清单

| 页面 | 路由 | 说明 |
|------|------|------|
| 登录/注册 | `/login` | 支持手机号+验证码登录 |
| 首页 Dashboard | `/` | 资产总览、持仓列表、收益趋势 |
| 基金搜索结果 | `/search?q=xxx` | 搜索结果列表 |
| 基金详情 | `/fund/:code` | 基金完整信息展示 |
| 我的持仓 | `/portfolio` | 持仓管理主页 |
| 添加持仓 | `/portfolio/add` | 添加/编辑持仓表单 |
| 交易记录 | `/portfolio/records` | 交易流水 |
| 收益分析 | `/analysis` | 收益统计与分析图表 |
| 自选基金 | `/watchlist` | 自选基金管理 |
| 基金排行 | `/ranking` | 基金排行榜 |
| 基金筛选 | `/screener` | 多维度筛选 |
| 指数估值 | `/valuation` | 指数估值表 |
| 工具箱 | `/tools` | 计算器、对比等工具 |
| 个人中心 | `/settings` | 账号与偏好设置 |

### 5.3 配色方案

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色调 | `#1677FF` | 品牌蓝，用于主要按钮、导航高亮 |
| 上涨色 | `#F5222D` | 红色，表示上涨/盈利（符合中国用户习惯） |
| 下跌色 | `#52C41A` | 绿色，表示下跌/亏损 |
| 背景色 | `#F5F7FA` | 浅灰，页面背景 |
| 卡片色 | `#FFFFFF` | 白色，内容卡片背景 |
| 文字主色 | `#1F2937` | 深灰，正文文字 |
| 文字辅色 | `#6B7280` | 中灰，辅助说明文字 |

### 5.4 关键交互说明

1. **金额隐藏**：点击"眼睛"图标切换金额显示/隐藏，隐藏时显示为 `****`
2. **下拉刷新**：移动端支持下拉刷新估值数据
3. **图表交互**：所有图表支持触摸/鼠标悬浮查看具体数据点
4. **搜索联想**：输入2个字符即触发联想，展示基金名称和代码
5. **空状态引导**：新用户无持仓时展示引导页，引导添加第一只基金
6. **加载状态**：数据加载时展示骨架屏（Skeleton），避免页面闪烁

## 七、数据需求

### 7.1 基金数据源

| 数据 | 来源 | 更新频率 |
|------|------|---------|
| 基金基本信息 | 天天基金/东方财富API | 每日 |
| 基金净值 | 天天基金/东方财富API | 每交易日收盘后 |
| 基金估值 | 第三方估值接口 | 交易日盘中实时 |
| 基金经理信息 | 公开数据 | 每季度 |
| 基金持仓数据 | 基金季报/半年报 | 每季度 |
| 指数估值数据 | 交易所/Wind | 每交易日 |

### 7.2 核心数据模型

```
用户（User）
├── id, phone, password_hash, nickname, avatar
├── created_at, updated_at

账户（Account）
├── id, user_id, name, platform, icon
├── created_at

持仓（Position）
├── id, user_id, account_id, fund_code
├── shares, cost_amount, cost_price
├── created_at, updated_at

交易记录（Transaction）
├── id, user_id, position_id, fund_code
├── type (buy/sell/dividend)
├── amount, shares, price, fee
├── trade_date, created_at

自选基金（Watchlist）
├── id, user_id, fund_code, group_name
├── created_at

基金信息（Fund）
├── code, name, type, company
├── manager, establishment_date, scale
├── risk_level, fee_rate
├── updated_at

基金净值（FundNav）
├── fund_code, nav_date
├── nav, accumulated_nav, daily_return

指数估值（IndexValuation）
├── index_code, index_name, date
├── pe, pb, roe, pe_percentile, pb_percentile
```

## 十、风险与约束

| 风险项 | 说明 | 应对策略 |
|--------|------|---------|
| 数据源稳定性 | 依赖第三方基金数据接口 | 多数据源备份，本地数据缓存 |
| 数据准确性 | 估值数据可能与实际净值有偏差 | 明确提示"仅供参考"，及时更新实际净值 |
| 合规风险 | 不得提供投资建议 | 所有分析标注"仅供参考，不构成投资建议" |
| 用户隐私 | 持仓数据属于敏感信息 | 数据加密存储，严格访问控制 |
